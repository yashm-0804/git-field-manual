# Git & GitHub — Field Manual

A single-page, self-contained reference for Git and GitHub: the whole workflow laid out so you can find the command you need fast and understand *why* it works, not just what to type.

No build step, no dependencies. One HTML file. Open it and go.

## 📖 What's inside

- **The mental model** — how Git actually thinks (working tree, staging, commits, HEAD)
- **Setup & identity** — first-time config that saves you headaches later
- **Start a project** — `init`, `clone`, and the first commit
- **The daily cycle** ★ — the add → commit → push loop you'll live in
- **Branches & merging** — creating, switching, merging, resolving conflicts
- **Remotes & sync** — `fetch`, `pull`, `push`, and keeping forks current
- **Undoing mistakes** — `restore`, `reset`, `revert`, and when to use which
- **Inspecting history** — `log`, `diff`, `blame`, `show`
- **Stashing** — parking work in progress safely
- **Forks & pull requests** — the open-source / team contribution flow
- **.gitignore** — patterns that matter
- **Common recipes** — copy-paste fixes for everyday situations
- **Cheat sheet** — the commands, condensed
- **Glossary** — plain-English definitions

## 🚀 View it

**Locally:** clone the repo and open `index.html` in any browser.

```bash
git clone https://github.com/<your-username>/git-field-manual.git
cd git-field-manual
open index.html      # macOS  ·  use "start" on Windows, "xdg-open" on Linux
```

**Live (GitHub Pages):** enable Pages in repo **Settings → Pages → Branch: `main` / root**, then visit:

```
https://<your-username>.github.io/git-field-manual/
```
